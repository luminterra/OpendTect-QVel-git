Intro

With this open-source plug-in you can build a quick post-stack seismic interval velocity model from T/D curves or Well Markers, Horizons, and optionally merged with Stacking Velocity cubes. The algorithm computes a velocity point from Horizon time values and corresponding depths (from either T/D curves or well markers) for every well/horizon intersection. Working from the top down, the algorithm grids all of these calculated velocity values layer-by-layer to generate a velocity field between adjacent time horizons. If a horizon is incomplete anywhere, the previous horizon is used to define the top of the interval. The algorithm is well-behaved with horizon pinch-outs and "zero thickness" horizon intervals. The bottom interval below the bottom horizon is simply an extension of the last interval's velocity. If input stacking velocities are used, the average interval velocity for the stacking velocity field is adjusted by the computed interval velocity. In this way stacking velocity character can be preserved (especially away from well control) while at the same time the velocity values from well control are preserved.

Another method of building a velocity model is to use all of the information in T/D curves, optionally with stratigraphic conformance with input horizons. With no horizons selected the algorithm horizontally grids every z slice using all available T/D curve data. Only 1/r gridding is available with this method. NOTE: this can be quite slow with seismic (< 20ms) z sampling. If horizons are selected as input then the gridding and extrapolation is constrained on an interval-by-interval basis so that horizon structure between wells is preserved in the gridded velocity field.

From the OpendTect menu, select


Analysis --> QVel Velocity Model Builder

Volume Subselection

In the Volume Subselection field you can select the inline/crossline extents and increments, and the Z (Time) extents and increments. The Z (time) increment limits the vertical resolution of the model. QVel velocity models must start at t=0. The model can be larger than the input seismic. For example, if you want to process just one line of stacking velocities, you should still build a 3D model with enough extent that it encompasses all the wells of interest.

Input Cube

For the Input Cube selection field, select an optional existing 3D time seismic cube. If the cube is a finely sampled Interval Velocity (also known as Instantaneous Velocity) volume, it will be merged with the constructed model (below). If your Stacking velocity cube is of another type (e.g, RMS), first convert the cube to type Interval (Processing --> Create Seismic Output --> Velocity Conversion). If the cube isn't an Interval Velocity cube, the only use of the volume will be to establish the sample rate and Inline/Crossline extents of the computed model. The input cube can be smaller than the model. If the input cube is larger than the model the output cube will be limited to the model's inline/crossline extent.

Select Horizons


Select Horizons lets you choose the time horizons with which you will build the model. Horizons can be incomplete, and they may "pinch out" against other horizons. Horizons must not cross in a nontrivial way or incorrect results will occur. If you are using Well Markers as depth control, every horizon selected must have at least one matching well marker.

Calibrate Input Stacking Velocities


Calibrate Input Stacking Velocities is enabled only if an input interval velocity volume is selected. You can then choose as to how differences in the layer-by-layer interval velocities between  the computed model and the input stacking velocities are handled.  Choose "By Entire Interval" if you want the stacking velocities to be scaled over the entire layer interval so that the net average interval velocity for the stacking velocity samples equals that of the computed interval average velocity.  Choose "By Well Weights" if you want the stacking velocities to be scaled to match the computed velocity only near wells. The interval velocity values away from well control will smoothly blend to the original stacking velocity values, based upon the 1/R gridding parameters (below).

Select Wells


In the Select Wells field, select one or more wells to be used in calculating the layer interval velocities between horizons.

Horizontal Interval Velocity Depth Control

You can choose between using Time/Depth curve data for calculating layer interval velocities or using Marker depth values for the corresponding time horizons. Rename Markers as necessary to match the corresponding Horizon names.

Fine Velocity Structure

You can choose between using Time/Depth curve data for calculating finely layered horizon-conformant velocities or by using seismic stacking or other finely structured interval velocity volume input. If no stacking velocities are available for input then no inter-horizon structure will be created and only average interval velocity between layers will be produced. The velocities between layers will in both cases vary laterally based on the well control and horizontal gridding parameters.

Velocity Gridder

You can choose to use a "thin plate spline" surface fit to fill the horizon interval with calculated velocity values at well/horizon intersections, or you can choose a modified 1/R surface fit. The spline fit will tend to extrapolate the velocites beyond the data points in the direction of the local curve fit, whereas 1/R type gridders tend to revert to the mean velocity away from well locations. Thus, the spline fit will tend to smoothly follow trends, and the 1/R type gridder will tend to create localized pools of velocity values near wells.

Spline Flatness

The spline flatness entry controls the amount of flatness applied to the spline fit (if selected). A value of zero forces the surface fit to honor every data point at the well/horizon intersections. It might show some polynomial ringing in the velocity grid. As the flatness value is increased to values between 1--5, the restriction that data points be honored is increasingly relaxed in favor of dampening the oscillation of the surface fit to make it approach a tilted flat plane

Well Influence Radius

This value (in inline/crossline steps) established the normalization value for the "R" in a 1/R type gridding. A larger value will spread the area of maximum influence for the gridding away from the well. Larger values tend to reduce the narrow "bullseye" effect of 1/R style gridders. The well influence Radius and Power (below) are used if there aren't enough valid points for a Spline surface gridder (if chosen), and are also used for the Calibrate stacking velocities by Well Weights. For example, if you choose 1/R, with an R of 10, then 10 lines/traces away from the well the velocity contribution to the interval grid is 1/(R/10), that is, still 100%. The conribution is never greater than 100%.

1/R^^N Power

This value is the exponent used in the 1/R^^N gridding calculation. A value of 1 results in a 1/R gridder. A value of 2 results in a 1/R-squared gridding which will cause the well velocity to fall away more quickly to the mean of all velocities away from the well.

Output Cube

 Select the name of the resulting output Interval Velocity cube. This cube is suitable for Time/Depth conversions.
